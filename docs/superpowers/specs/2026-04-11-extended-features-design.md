## 1. 目标

在 MVP 基础上扩展 7 个功能模块，把单按钮「今日冒险」扩展成完整的闯关+收集+社交体验：

1. **果园探险地图** — 20 站点分支路径，星级评价
2. **果园装饰系统** — 53 种装饰物，星星兑换，自由摆放
3. **水果收集图鉴** — 30 种水果，关卡奖励解锁
4. **数字火车** — 凑十法玩法（新 SKScene）
5. **天平乐园** — 等式概念玩法（新 SKScene）
6. **L5/L6 难度** — 20 以内加减法含进退位
7. **亲子对战** — iPad 横屏双向分屏 5 轮对决

不做：多孩档案（用户明确表示不需要）

enum DifficultyLevel: Int, ... {
    case seed = 1       // L1: 5 以内加法
    case sprout = 2     // L2: 5 以内加减法
    case smallTree = 3  // L3: 10 以内加法
    case bigTree = 4    // L4: 10 以内加减法

    case harvest = 6    // L6: 20 以内加减法 (新，含进退位)

    var maxNumber: Int {

        case .seed, .sprout: return 5
        case .smallTree, .bigTree: return 10
        case .bloom, .harvest: return 20  // 新增
        }
    }

    var allowsSubtraction: Bool {
        switch self {
        case .seed, .smallTree, .bloom: return false
        case .sprout, .bigTree, .harvest: return true
        }
    }

## 4. 新玩法详细设计

### 4.1 数字火车 (凑十法)

**教学目标:** 理解数的分解组合，凑十法。

**场景:**
```
[题目文字 + 语音]

🚂 ┌──┬──┬──┬──┬──┬──┬──┬──┬──┬──┐
🚂 │🐻│🐰│🐸│🐶│🐱│🐷│  │  │  │  │
   └──┴──┴──┴──┴──┴──┴──┴──┴──┴──┘
    ●  ●  ●  ●  ●  ●  ○  ○  ○  ○

         答案: [ _ ]

    [1] [2] [3] [4] [5]
    [6] [7] [8] [9] [0]
```

**两种交互模式（按级别自动切换）:**

| 模式 | 级别 | 交互 |
|------|------|------|
| 点数模式 | L2-L3 | 逐个点空座位，每点计数，自动填入答案 |
| 输入模式 | L4+ | 数字键盘输入 |

**题型矩阵:**

| 题型 | 算式示例 | 级别 |
|------|---------|------|
| 求空位 | `6 + ? = 10`（已坐 6，求空） | L2-L3 |
| 求已坐 | `? + 4 = 10`（空 4，求已坐） | L2-L3 |
| 5 的分解 | `a + b = 5`（开放） | L2 |
| 10 的分解 | `a + b = 10`（开放） | L4 |
| 双车厢进位加 | `9 + 3 = ?`（利用凑十） | L5-L6 |

**数据产出:** 答题结果走与摘果子/分果果相同的 `handleAnswer()`。

**完成反馈:**
- 🚂💨 火车鸣笛
- 空位依次跳入小动物
- 火车驶出右侧
- 下一题火车驶入左侧

### 4.2 天平乐园 (等式概念)

**教学目标:** 理解等号 `=` 是"两边一样多"，不是"答案是"。

**场景:**
```
      "让天平平衡吧！"

             △
     ┌───────┴───────┐
   ↙                   ↘  (实时倾斜)
┌─┴─┐               ┌─┴─┐
│🟦🟦🟦│    =       │🟦🟦+?│
│ 5  │               │ 2+? │
└────┘               └────┘

积木池: 🟦🟦🟦🟦🟦🟦🟦🟦 (可拖)
```

**物理反馈:**
```
倾斜角度 = (左侧重量 - 右侧重量) × 5°，夹在 ±30°

放积木:
  - 弹性落入 0.2s
  - 天平平滑过渡 0.3s
  - 小幅晃动 0.5s 后稳定

两边相等:
  - 天平水平 + 金色发光
  - 两托盘连起彩虹桥
  - 浮现等式："5 = 2 + 3"
  - 平衡音效 + 语音鼓励
```

**可逆操作:**
- 可从托盘拖走积木（理解减法与加法关系）
- 可拖回积木池
- 天平实时响应，无唯一路径

**题型:**

| 题型 | 示例 | 级别 |
|------|------|------|
| 补全右侧 | `5 = 2 + ?` | L2 |
| 补全左侧 | `10 = ? + 3` | L4 |
| 两边都填 | `? + 4 = 3 + ?` (多解) | L5 |
| 大数平衡 | `15 = 8 + ?` | L6 |

### 4.3 L5/L6 难度扩展

**L5 (20 以内加法):**
- 不进位: 11+3, 12+5, 14+5 (个位相加 < 10)
- 进位: 8+5, 9+6, 7+8 (个位相加 ≥ 10)

**L6 (20 以内加减法):**
- L5 全部 +
- 不退位减法: 15-3, 18-5
- 退位减法: 13-5, 12-7, 15-9

### 4.4 四玩法在站点中的分配

| 站点级别 | 主要玩法 | 辅助玩法 |
|---------|---------|---------|
| L1 纯加法 | 摘果子 | 数字火车(点数模式) |
| L2 加减 | 摘果子/分果果 | 数字火车 |
| L3 纯加法 | 摘果子 | 数字火车/天平 |
| L4 加减 | 摘果子/分果果 | 数字火车/天平 |
| L5 20以内加 | 数字火车/天平 | 摘果子 |
| L6 加减进退位 | 数字火车/天平 | 分果果 |

每站 5 题至少混合 2 种玩法，避免疲劳。

## 8. 亲子对战

### 8.1 入口保护

需要通过 Parental Gate（避免孩子误入，因为需要成人参与）。

### 8.2 UI 布局（iPad 横屏放平桌上）

```
┌─────────────────────────────────────────────────┐
│ ┌─────────────────────────────────────────────┐ │
│ │        👨 家长区域 (UI 旋转 180°)             │ │
│ │                                              │ │
│ │        47 × 3 = [___]    ← 家长题              │ │
│ │                                              │ │
│ │      [1][2][3][4][5]                          │ │
│ │      [6][7][8][9][0]                          │ │
│ ├═══ 🏆 比分 孩子 2 : 1 家长 ═════════════════┤ │
│ │                                              │ │
│ │       3 + 2 = [___]   ← 孩子题                 │ │
│ │                                              │ │
│ │      [1][2][3][4][5]                          │ │

│ └─────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────┘
```

### 8.3 规则

```
赛制: 5 轮对决

每轮：
  - 双方同时出题
  - 孩子题: 当前学习级别 (L1-L6)
  - 家长题: 孩子级别 × 5 的难度
  - 谁先正确作答 → 得 1 分

难度对应:
  L1 (5内) → 家长做 25 内加减
  L2 → 25 内混合
  L3 (10内) → 50 内加减
  L4 → 50 内混合带乘除
  L5 (20内) → 100 内混合
  L6 → 100 内混合带乘除
```

### 8.4 动态平衡

```
IF 孩子连输 2 轮:
    家长下一题难度 × 1.5
    家长输入按钮缩小 20%

IF 家长连输 2 轮:
    家长下一题难度 × 0.8 (避免过假)
```

### 8.5 结算

- 孩子赢：烟花 + 奖杯 + 语音"你比爸爸/妈妈还厉害！"
- 家长赢：拥抱动画 + "差一点就赢了，下次一定！"
- 平局：击掌 + "你们都很棒！"
- **不记录胜负历史**（不存 DB）

### 8.6 技术要点

- SwiftUI `.rotationEffect(.degrees(180))` 翻转家长区域
- 多点触控 (同时响应两个触点)
- 防误触：上下区域间 40pt 中性带
- 不与 ChildProfile 的 LearningSession 关联（单独的 battle state）

## 9. 数据流

### 9.1 端到端数据流

```
孩子答题 (四种 Scene 之一)
    │
    ▼
AdventureSessionViewModel
    │ ├─ 记录到 session.records (现有)
    │ ├─ 更新 LearningProfile (现有)
    │ ├─ 新增: 更新 StationProgress (stationId, stars, accuracy)
    │ └─ 计算奖励 (stars/seeds/fruit)
    ▼
ChildProfile (SwiftData) 持久化
    │ ├─ profile.stars += N
    │ ├─ profile.seeds += N
    │ ├─ 插入 CollectedFruit 如有
    │ ├─ 触发解锁下一站
    │ └─ 更新 treeStage 如升级
    ▼
SwiftUI 自动响应式刷新
    ├── HomeView (装饰、果树、星星数)
    ├── ExplorationMapView (星级)
    ├── FruitCollectionView (新水果)
    └── DecorateOrchardView (星星数)
```

### 9.2 关键决策点

| 场景 | 逻辑 |
|------|------|
| 完成 ★ | stars +3, seeds +1, 30% 随机常见水果 |
| 完成 ★★ | 额外 stars +2 |
| 完成 ★★★ | 额外 stars +3, 稀有水果 1 |
| 完成 L6 任一 ★★★ | 解锁传说水果 + 终点果园 |
| 果树升级 | seeds × 当前阶段 被消耗 |

## 13. 成功标准

实现完成后，用户体验目标：

1. 孩子打开 app，看到装饰好的果园，感受到自己的"成果"
2. 点击地图，看到多个待挑战关卡，星级激励驱动刷关
3. 完成站点获得水果→图鉴填充
4. 累积星星→买装饰→果园变化
5. 想难一点可打 L5/L6（凑十法+进退位）
6. 想跟家长玩 → 亲子对战模式
7. 4 种玩法轮换，无单一疲劳

孩子有清晰的"为什么要玩"的回答：闯关、收集、装饰、对战。
