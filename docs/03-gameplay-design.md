# 数字果园 — 玩法设计详细文档

> 版本: v1.0
> 日期: 2026-04-11

---

## 1. 设计理念

### 1.1 核心原则

1. **做透不做广** — 只做 20 以内加减法，但用 6 种玩法从不同维度让孩子真正理解
2. **具象先行** — 每一道算式都有对应的实物操作，先"摸到"再"算到"
3. **温和激励** — 没有失败，只有"再来一次"；没有惩罚，只有鼓励
4. **自然节奏** — 每天 15-20 分钟，像每天吃一个苹果一样自然

### 1.2 教学法基础 — CPA 三阶段

```
C (Concrete 具象)     →    P (Pictorial 图示)    →    A (Abstract 抽象)
拖拽真实水果              看图片数水果                   纯数字算式

示例（3 + 2 = 5）:

C: 篮子里3个苹果，从树上拖2个进去，数一数变成5个
P: 看到3个苹果图片和2个苹果图片，中间有"+"号
A: 屏幕显示 "3 + 2 = ?" ，孩子直接输入5
```

**在 App 中的实现：**
- L1-L2（5以内）：主要使用 C 阶段，大量实物操作
- L3-L4（10以内）：C + P 混合，逐步引入图示
- L5-L6（20以内）：P + A 混合，偶尔回到 C 阶段巩固

## 2. 六大玩法详细设计

### 2.1 摘果子 — 加法理解

#### 学习目标
- 理解加法是"把两组东西合在一起"
- 建立"部分 + 部分 = 整体"的心理模型
- 从数实物过渡到心算

#### 场景布局（iPad 横屏）

```
┌────────────────────────────────────────────────────────┐
│  ┌─────────────────────────────────────────────────┐   │
│  │        "篮子里有 3 个，再摘 2 个，一共几个？"       │   │
│  │              🔊 (点击重新播放语音)                 │   │
│  └─────────────────────────────────────────────────┘   │
│                                                        │
│       🍎                                               │
│     🍎  🍎            ┌──────────┐                     │
│    🌳🌳🌳🌳           │  🍎🍎🍎  │                     │
│    🌳🌳🌳🌳           │   [3]    │                     │
│   🌳🌳🌳🌳🌳          │  篮子    │                     │
│                       └──────────┘                     │
│                                                        │
│                    ── 进度 2/5 ──                       │
└────────────────────────────────────────────────────────┘
```

#### 交互细节

**拖拽行为：**
- 手指触摸水果时：水果放大 120%，轻微弹起，播放"嘭"音效
- 拖拽过程中：水果跟随手指，下方出现半透明投影
- 靠近篮子时：篮子边缘发光提示"可放入"
- 松手在篮子区域：水果弧线落入篮子，弹跳两下，数字 +1
- 松手在非目标区域：水果滑回原位

**答对流程（时间轴）：**
```
T+0.0s  最后一个水果落入篮子
T+0.2s  篮子里所有水果同时弹跳一下
T+0.5s  上方算式补全 "3 + 2 = 5" (数字从小变大动画)
T+0.8s  星星粒子从篮子中心向四周爆射
T+1.0s  语音播报 "太棒了！三加二等于五！"
T+1.5s  经验值 "+10" 浮字上飘消失
T+2.0s  过渡到下一题（水果树刷新）
```

**答错处理：**
```
多拖了水果：
    - 篮子轻微左右摇晃 (0.3s)
    - 柔和提示音（不是刺耳的"错误"音）
    - 语音："好像多了一个哦，我们数数篮子里有几个？"
    - 多余水果自动弹回树上

少拖了就点确认（如有确认按钮的变体）：
    - 语音："好像还不够呢，树上还有苹果可以摘哦"
    - 树上剩余水果轻微摇晃吸引注意
```

#### 难度递进表

| 子难度 | 算式范围 | 辅助手段 | 示例 |
|--------|---------|---------|------|
| L1-1 | 1+1, 1+2, 2+1 | 水果有数字标签，篮子有数字 | 🍎① + 🍎② |
| L1-2 | 1+3, 2+2, 3+1 | 水果有数字标签 | |
| L1-3 | 2+3, 3+2, 1+4 | 无标签，需自己数 | |
| L3-1 | 4+1 ~ 5+2 | 无标签 | |
| L3-2 | 3+4 ~ 5+5 | 引入十格框辅助 | |
| L5-1 | 6+5 ~ 8+4 (不进位) | 十格框 | |
| L5-2 | 7+6 ~ 9+8 (进位) | 两个十格框 | |

### 2.2 分果果 — 减法理解

#### 学习目标
- 理解减法是"从一组里拿走一部分"
- 建立"整体 - 部分 = 剩余"的心理模型
- 体会"分享"的社交含义（减法不是"失去"而是"给予"）

#### 场景布局

```
┌────────────────────────────────────────────────────────┐
│  ┌─────────────────────────────────────────────────┐   │
│  │      "盘子里有 8 个草莓，分给小兔 3 个，剩几个？"   │   │
│  └─────────────────────────────────────────────────┘   │
│                                                        │
│              ┌───────────────┐                          │
│              │ 🍓🍓🍓🍓       │                          │
│              │ 🍓🍓🍓🍓  [8]  │                          │
│              └───────────────┘                          │
│                                                        │
│     🐰           🐻           🐵                       │
│    [盘子]       [盘子]       [盘子]                     │
│   (目标:3)     (不参与)     (不参与)                    │
│                                                        │
└────────────────────────────────────────────────────────┘
```

#### 交互细节

- 只有题目中指定的动物是可交互目标（高亮），其余动物灰色
- 每拖出一个水果，大盘子数字 -1，动画实时反馈
- 动物接到水果后做开心的小动画（兔子耳朵竖起来、猴子拍手）
- **完成判定**：当拖给目标动物的数量 = 题目要求时，自动触发结算（无确认按钮）
- 多拖水果给动物：动物摇头，水果弹回盘子
- 拖给非目标动物：该动物摆手拒绝，目标动物跳动吸引注意
- 分完后动物吃水果的萌系动画（1.5s）

#### 变体玩法（高级别解锁）

**多目标分配（L4+）：**
```
"8 个苹果，分给小兔 3 个，再分给小熊 2 个，还剩几个？"
→ 两步减法：8 - 3 - 2 = 3
→ 需要先给兔子，再给小熊（顺序自由）
```

**比较型（L4+）：**
```
"小兔有 5 个草莓，小猴有 3 个，小兔比小猴多几个？"
→ 用视觉对齐两排水果，凸显差异部分
→ 引入减法的"比较"含义
```

### 2.3 数字火车 — 凑十法

#### 学习目标
- 掌握"凑十"——10 的分解与组合
- 为 20 以内进位加法和退位减法打基础
- 理解未知数的概念（? + 6 = 10）

#### 场景布局

```
┌────────────────────────────────────────────────────────┐
│            "火车有 10 个座位，坐了 6 个，空几个？"        │
│                                                        │
│    ┌──┬──┬──┬──┬──┬──┬──┬──┬──┬──┐                    │
│    │🐻│🐰│🐸│🐶│🐱│🐷│  │  │  │  │ ← 火车车厢        │
│    └──┴──┴──┴──┴──┴──┴──┴──┴──┴──┘                    │
│     ●  ●  ●  ●  ●  ●  ○  ○  ○  ○                     │
│    (实心=已坐  空心=空位)                                │
│                                                        │
│              ┌─────────────┐                            │
│              │  输入答案: [_]│                            │
│              └─────────────┘                            │
│         ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐                          │
│         │1│ │2│ │3│ │4│ │5│    ← 数字按钮              │
│         └─┘ └─┘ └─┘ └─┘ └─┘                          │
│         ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐                          │
│         │6│ │7│ │8│ │9│ │0│                            │
│         └─┘ └─┘ └─┘ └─┘ └─┘                          │
└────────────────────────────────────────────────────────┘
```

#### 交互方式（双模式）

**模式 A — 点数模式（L2-L3，适合初学）：**
- 孩子可以逐个点击空座位，每点一下座位变亮并计数
- 点完后数字自动填入答案框
- 帮助建立"数一数"的习惯

**模式 B — 直接输入模式（L4+）：**
- 孩子直接点击数字键盘输入答案
- 训练心算能力

**答对后：**
- 火车鸣笛 🚂💨
- 空位上跳入可爱的小动物
- 火车缓缓驶出画面右侧
- 新火车从左侧驶入（下一题）

#### 题型变化

| 题型 | 示例 | 级别 |
|------|------|------|
| 求空位 | 10 个位，坐了 7 个，空 ? | L2 |
| 求已坐 | 10 个位，空了 4 个，坐了 ? | L3 |
| 5 的分解 | 5 个位，坐了 2 个，空 ? | L2 |
| 开放式 | 10 = ? + ?（多种答案都对） | L4+ |
| 双车厢 | 前车厢 6 个 + 后车厢 ? 个 = 10 | L4 |

### 2.4 天平乐园 — 等式概念

#### 学习目标
- 理解等号 "=" 的含义是"两边一样多"（而不是"答案是"）
- 建立等式平衡的直觉
- 为未来的代数思维奠基

#### 场景布局

```
┌────────────────────────────────────────────────────────┐
│              "让天平平衡吧！"                            │
│                                                        │
│                    △                                    │
│              ┌─────┴─────┐                              │
│           ┌──┘           └──┐                           │
│        ┌──┴──┐          ┌──┴──┐                        │
│        │🧊🧊🧊│          │🧊🧊 ?│                        │
│        │  5  │          │ 2+? │                        │
│        └─────┘          └─────┘                        │
│                                                        │
│        物品堆: 🧊🧊🧊🧊🧊🧊🧊🧊                         │
│        (可拖拽的积木)                                    │
└────────────────────────────────────────────────────────┘
```

#### 物理反馈系统

```
天平倾斜角度 = (左侧重量 - 右侧重量) × 5°
最大倾斜角度: ±30°

放上积木: 
    - 弹性落入动画 (0.2s)
    - 天平平滑过渡到新角度 (0.3s)
    - 轻微晃动后稳定 (0.5s)

平衡时:
    - 天平水平 + 发光
    - 彩虹桥连接两个托盘
    - 等式浮现："5 = 2 + 3"
```

#### 创新点：可逆操作

- 孩子可以把右侧已有的积木拖走（理解减法与加法的关系）
- 天平实时响应，鼓励自由探索
- 没有唯一正确的操作路径

### 2.5 果园探险 — 综合闯关

#### 学习目标
- 混合运用加减法，建立运算流畅性
- 适应不同题目呈现形式的切换
- 通过重复练习达到自动化

#### 地图设计

```
                    ★ 终点果园
                   / \
                 /     \
           [L5站点] [L6站点]
              |    ×    |
           [L4站点] [L4站点]
              \       /
               [L3站点]
              /       \
           [L2站点] [L2站点]
              \       /
               [L1站点]
                  |
               🏠 起点
```

- 每个站点包含 5 道题，题型从当前级别随机混合
- 分支路径允许孩子选择不同的探险路线
- 每个站点可反复挑战（刷星）

#### 星级评价系统

```
★☆☆  完成所有题目即可获得
★★☆  正确率 >= 80%（最多错 1 题）
★★★  正确率 100% 且未使用提示

星级只升不降：
    重玩时如果获得更高星级则更新
    不会因为重玩失败降低已有星级
```

#### 站点间过渡动画

- 完成站点后，孩子的小角色沿路径走向下一站
- 路径上散落着可收集的星星（自动拾取）
- 到达新站点时，小屋亮灯 + 门打开的动画

### 2.6 亲子对战 — 多人模式

#### 学习目标
- 在社交情境中激发学习动机
- 让"做数学"成为亲子间的快乐记忆
- 通过适当竞争提升反应速度

#### 分屏布局

```
┌────────────────────────────────────────────────────────┐
│ ┌────────────────────────────────────────────────────┐ │
│ │                 家长区域（倒置显示）                  │ │
│ │                                                    │ │
│ │         47 × 3 = [___]    ← 家长题目（难）          │ │
│ │                                                    │ │
│ ├────────────────────────────────────────────────────┤ │
│ │  ═══════ 🏆 比分 孩子 2 : 1 家长 ═══════            │ │
│ ├────────────────────────────────────────────────────┤ │
│ │                                                    │ │
│ │          3 + 2 = [___]   ← 孩子题目（适龄）         │ │
│ │                                                    │ │
│ │          [1][2][3][4][5]                            │ │
│ │          [6][7][8][9][0]                            │ │
│ └────────────────────────────────────────────────────┘ │
└────────────────────────────────────────────────────────┘
```

#### 平衡机制详细设计

```
家长题目难度 = 孩子当前级别 × 5

示例：
    孩子 L1 (5以内加法)  → 家长做 25以内加减法
    孩子 L3 (10以内加法) → 家长做 50以内乘除法
    孩子 L5 (20以内加法) → 家长做 100以内混合运算

动态平衡：
    IF 孩子连输 2 轮:
        家长下一题难度 × 1.5
        家长输入键变小 20%（物理增加输入难度）
    IF 家长连输 2 轮:
        家长下一题难度 × 0.8（避免太假）
```

#### 对战流程

```
1. 倒计时 3-2-1 开始
2. 双方同时出题
3. 谁先正确作答，中间区域向对方推进 + 得 1 分
4. 5 轮后结算
5. 结算动画：
   - 孩子赢：烟花 + 奖杯 + "你太厉害了！"
   - 家长赢：拥抱动画 + "差一点就赢了，下次加油！"
   - 平局：击掌动画 + "你们都很棒！"
```

## 5. 用眼保护机制

### 5.1 时间控制

```
第一档（软提醒）：达到设定时长的 80%
    - 屏幕右上角出现小时钟图标
    - 不打断当前操作

第二档（温和提醒）：达到设定时长
    - 当前题目完成后弹出全屏提醒
    - "小果农休息一下吧！站起来看看窗外～"
    - 倒计时 60 秒后可选择"再玩 5 分钟"或"结束"
    - "再玩 5 分钟"最多使用 1 次

第三档（强制锁定）：超出设定时长 5 分钟
    - 今日冒险入口锁定
    - 仍可浏览果园（不含算题）
    - 解锁需家长通过 Parental Gate
```

### 5.2 视觉保护

- 背景色避免纯白，使用暖色调米白 (#FFF8E7)
- 不使用闪烁/频闪动画
- 文字和数字使用高对比度但不刺眼的配色
- 夜间模式：晚间 20:00 后自动降低屏幕亮度对比度（可关闭）
