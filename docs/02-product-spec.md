## 1. 产品概述

### 1.1 产品名称

**数字果园** (Number Orchard)

### 1.2 一句话定位

专为 5-6 岁中国儿童设计的加减法冒险游戏，在 iPad 上用"玩"的方式建立数感。

### 1.3 目标用户

| 用户角色 | 画像 |
|---------|------|
| 主要用户 | 5-6 岁学龄前/幼小衔接儿童 |
| 付费决策者 | 儿童家长（25-40 岁） |
| 使用场景 | 家庭场景，iPad 横屏使用，每日 15-20 分钟 |

### 1.4 核心价值主张

| 竞品痛点 | 我们的解法 |
|---------|-----------|
| iPad 体验是手机放大版 | iPad 横屏原生设计，大屏触控优先 |
| 知识面广但加减法做不透 | 聚焦 20 以内加减法，做深做透 |
| 孩子会算但不理解为什么 | 实物化操作 + CPA 教学法，从具象到抽象 |
| 家长只能看后台报告 | 亲子对战/协作模式，家长参与学习 |
| 刷题枯燥或游戏沉迷 | 故事驱动 + 节奏控制（15-20 分钟/天） |

### 2.2 自适应难度引擎

#### 2.2.1 六级难度体系

| 级别 | 名称 | 运算范围 | 解锁条件 |
|------|------|---------|---------|
| L1 | 种子 | 5 以内加法 | 初始解锁 |
| L2 | 发芽 | 5 以内加减法 | L1 正确率 >= 80% |
| L3 | 小树 | 10 以内加法 | L2 正确率 >= 75% |
| L4 | 大树 | 10 以内加减法 | L3 正确率 >= 75% |
| L5 | 开花 | 20 以内加法（不进位） | L4 正确率 >= 70% |
| L6 | 结果 | 20 以内加减法（含进退位） | L5 正确率 >= 70% |

#### 2.2.2 动态调整规则

```
IF 连续答对 >= 3 题:
    difficulty += 1（在当前级别内提升子难度）

IF 连续答错 >= 2 题:
    difficulty -= 1
    切换为辅助教学模式（显示实物操作帮助）

IF 当前级别正确率达标 AND 完成最低题量:
    解锁下一级别

评估指标:
    - 正确率（primary）
    - 反应时间（secondary）— 快速且正确额外加分
    - 辅助使用频率（secondary）— 频繁使用提示则减缓进阶速度
```

#### 2.2.3 题目生成规则

- 同一运算不连续出现超过 3 次（避免疲劳）
- 加法和减法交替出现（L2 及以上）
- 每 5 题至少包含 1 道"实物辅助题"（巩固概念理解）
- 答案尽量分散，避免连续出现相同答案

### 2.3 核心玩法详细规格

#### 玩法 1：摘果子（加法）

**教育目标：** 理解"加法就是合在一起"

**场景描述：**
- 画面左侧是一棵果树，上面挂着若干水果
- 画面右侧是一个篮子，里面已有若干水果
- 上方显示算式（带语音朗读）：「篮子里有 3 个苹果，再摘 2 个，一共多少个？」

**交互流程：**
```
1. 场景加载，语音播报题目
2. 孩子从树上逐个拖拽水果到篮子
3. 每拖入一个，篮子上方数字实时更新（3→4→5）
4. 拖入正确数量后：
   - 篮子发光 + 水果弹跳动画
   - 算式自动补全 "3 + 2 = 5"
   - 语音："太棒了！3 加 2 等于 5！"
5. 如果拖入过多：
   - 篮子轻微摇晃 + 温和提示音
   - 语音："好像多了哦，数一数篮子里有几个？"
   - 多余的水果自动弹回树上
```

**难度变化：**
| 级别 | 加数范围 | 辅助 |
|------|---------|------|
| L1 | 1+1 到 2+3 | 树上水果有数字标签 |
| L3 | 1+1 到 5+5 | 无标签，需自己数 |
| L5 | 5+1 到 10+10 | 引入"十格框"辅助 |

---

#### 玩法 2：分果果（减法）

**教育目标：** 理解"减法就是拿走/分出去"

**场景描述：**
- 画面中央是一个大盘子，里面有若干水果
- 画面下方有若干小动物（兔子、猴子、小熊等），各举着空盘子
- 上方算式：「盘子里有 8 个草莓，分给小兔 3 个，还剩几个？」

**交互流程：**
```
1. 语音播报题目
2. 孩子把水果逐个拖给目标小动物
3. 大盘子上方数字实时递减（8→7→6→5）
4. 分完指定数量后：
   - 小动物做吃水果的可爱动画
   - 算式补全 "8 - 3 = 5"
   - 语音："真棒！8 减 3 等于 5！"
5. 如果分的数量不对：
   - 小动物摇头 + 温和提示
   - 语音："再数一数盘子里还剩几个？"
```

**变体玩法（L4+）：**
- 多个动物同时需要分配（如：8 个分给 2 只动物各 3 个，剩？）
- 引入"比较"场景：谁分到的更多？

---

#### 玩法 3：数字火车（凑十法 / 数的分解）

**教育目标：** 理解凑十法，掌握数的分解与组合

**场景描述：**
- 画面是一列火车，共 10 节车厢
- 部分车厢已有小动物乘客，部分车厢空着
- 上方提问：「火车有 10 个座位，已经坐了 6 个小动物，还有几个空座？」

**交互流程：**
```
1. 火车从左侧驶入画面，停靠在站台
2. 可视化展示：已占座位（亮色）vs 空座位（灰色虚线）
3. 孩子点击数字键盘输入答案
   - 或者拖拽小动物填入空座位来"数一数"
4. 答对后火车鸣笛出发，驶向下一站
5. 答错后站台上的小瓢虫引导："让我们一起数数空座位吧！"
   - 空座位依次高亮闪烁，语音带着一起数
```

**难度变化：**
| 级别 | 题型 | 示例 |
|------|------|------|
| L2 | ? + n = 5 | ? + 3 = 5 |
| L4 | ? + n = 10 | ? + 6 = 10 |
| L4 | n + ? = 10 | 7 + ? = 10 |
| L6 | 数的分解 | 10 = ? + ? (开放式) |

---

#### 玩法 4：天平乐园（等式概念）

**教育目标：** 理解等号的含义——"两边一样多"

**场景描述：**
- 画面中央一个大天平
- 两侧可以放置水果/积木
- 天平根据两侧重量实时倾斜

**交互流程：**
```
1. 天平一侧已放置若干物品（如左边 5 个积木）
2. 右边已有部分物品（如 2 个），提示需要再加几个才能平衡
3. 孩子从旁边的物品堆拖拽积木到右侧
4. 每放一个，天平实时动画倾斜变化
5. 天平平衡时：
   - 天平发光 + 平衡彩虹动画
   - 算式显示 "5 = 2 + 3"
   - 语音："两边一样多了！5 等于 2 加 3！"
```

**物理反馈：**
- 天平倾斜角度与两侧差值成正比
- 放上物品有轻微弹性动画
- 天平底座有数字刻度辅助

**难度变化：**
| 级别 | 题型 |
|------|------|
| L2 | 5 = 2 + ? |
| L4 | 10 = ? + 3 |
| L5 | ? + 4 = 3 + ? (两边都需填) |
| L6 | 15 = 8 + ? |

---

#### 玩法 5：果园探险（综合闯关）

**教育目标：** 综合应用加减法，建立运算流畅性

**场景描述：**
- 俯视角果园地图，多条小路连接不同站点
- 每个站点是一个"果园小屋"，包含一组题目
- 完成当前站点解锁相邻站点

**关卡结构：**
```
每个站点：
    - 5 道题（混合题型，从当前级别题库随机）
    - 沙漏计时（装饰性，不强制中断）
    - 星级评价：
        ★☆☆  完成即获得
        ★★☆  正确率 >= 80%
        ★★★  正确率 = 100% 且无使用提示

奖励：
    - 每站必得：1 颗种子
    - 2 星+：1 个随机装饰品
    - 3 星：1 个稀有水果品种
```

**地图设计：**
- 初始地图包含 20 个站点（约 100 道题）
- 每个难度级别对应一片地图区域
- 支持重玩已通过站点（刷星）

---

#### 玩法 6：亲子对战（多人模式）

**教育目标：** 增强学习动机，建立亲子间积极的数学联想

**场景描述：**
- iPad 横屏，画面上下分为两个区域
- 上方：家长区域（倒置显示，面向家长）
- 下方：孩子区域（正向显示）

**对战规则：**
```
赛制：5 轮对决

每轮：
    - 双方各出一道题
    - 孩子题目：当前学习级别
    - 家长题目：难度远高于孩子（两位数运算/混合运算）
    - 谁先答对谁得 1 分

平衡机制：
    - 家长题目难度是孩子的 3-5 倍
    - 家长答题区域故意做小（增加输入难度）
    - 如孩子连输 2 轮，家长下一题难度进一步提升

结算：
    - 孩子赢：烟花动画 + "你比爸爸/妈妈还厉害！"
    - 家长赢：拥抱动画 + "下次一定能赢！"
    - 不记录胜负历史（避免挫败感累积）
```

**技术要求：**
- 多点触控支持两人同时操作
- 上方区域 UI 旋转 180 度
- 防误触：两个区域间有明确分隔带

### 2.4 果园系统（成长 & 收集）

#### 2.4.1 果树成长

```
经验值获取：
    - 答对 1 题：+10 EXP
    - 连续答对 3 题 combo：额外 +5 EXP
    - 完成每日任务：+50 EXP
    - 完成探险站点：+30 EXP

果树阶段：
    种子 (0 EXP) → 发芽 (100) → 小苗 (300) → 小树 (600)
    → 大树 (1000) → 开花 (1500) → 结果 (2000)

    每棵果树结果后，解锁下一棵新果树
    果园最多可种 12 棵果树
```

#### 2.4.2 水果收集

- 共 30 种水果（苹果、草莓、西瓜、芒果、火龙果……）
- 通过探险关卡 3 星获得稀有品种
- 收集图鉴，每种水果附带趣味小知识

#### 2.4.3 果园装饰

- 用星星兑换装饰物：栅栏、花丛、蝴蝶、小池塘、秋千……
- 纯装饰性，无付费道具，所有装饰均可通过游玩获得
- 装饰品总数 50+，持续更新

### 2.6 家长中心

**入口保护：** Parental Gate — 长按圆形按钮 3 秒 + 向右滑动解锁（防儿童误入）

#### 2.6.1 学习报告

```
每日报告：
    - 今日练习题数 / 正确率 / 用时
    - 答错题目回顾（带正确解法展示）
    - 当前级别进度条

每周报告：
    - 一周趋势图（正确率 & 题量）
    - 薄弱环节分析（如"5以内减法需加强"）
    - 与同龄儿童平均水平对比（脱敏匿名数据）
    - 建议：未来一周学习重点

成长记录：
    - 学习天数 / 总答题量 / 解锁成就
    - 里程碑时间线
```

#### 2.6.2 设置项

| 设置 | 说明 | 默认值 |
|------|------|--------|
| 每日使用时长上限 | 超时自动锁定 | 20 分钟 |
| 难度锁定 | 手动锁定/解锁特定级别 | 自适应 |
| 音效开关 | 背景音乐/音效/语音 独立控制 | 全开 |
| 语言 | 中文/英文 | 跟随系统 |
| 通知 | 每日学习提醒 | 开启 (18:30) |

#### 2.6.3 多孩档案

- 一个 Apple ID 下支持最多 4 个独立档案
- 每个档案独立的学习进度、难度级别、果园状态
- 档案切换需通过 Parental Gate
- 支持 iCloud 家庭共享（一人订阅全家可用）

## 3. UI/UX 规格

### 3.1 设计原则

| 原则 | 规格 |
|------|------|
| 横屏优先 | 仅支持横屏模式，不支持竖屏旋转 |
| 大触控区域 | 最小可点击区域 44×44pt，核心操作区域 >= 80×80pt |
| 无文字依赖 | 所有指引使用图标 + 语音朗读，不识字也能独立操作 |
| 色彩安全 | 主色调暖色系（果园绿 #4CAF50 + 苹果红 #FF5252 + 阳光黄 #FFD740），通过 WCAG AA 对比度检测 |
| 动画克制 | 操作反馈动画 <= 0.3s，奖励动画 <= 2s，不阻塞用户操作流 |
| 声音分层 | 背景音乐 / 音效 / 语音 三个独立音量控制 |
| 误触容错 | 拖拽需落入目标区域才确认；关键操作（如退出关卡）需二次确认 |

### 3.2 信息架构

```
App 启动
├── 档案选择（多孩时显示，单孩跳过）
└── 主界面（果园全景）
    ├── [左] 果园区域（果树 + 装饰，可自由浏览）
    ├── [中下] 今日冒险入口（大按钮，脉冲动画吸引注意）
    ├── [右上] 探险地图入口
    ├── [右下] 亲子对战入口
    └── [左上] 家长中心入口（小齿轮，Parental Gate）
```

### 3.3 关键界面规格

#### 主界面（果园全景）

- 全景横屏插画风格，日间/傍晚/夜间随真实时间变化
- 果树根据成长阶段展示不同形态
- 装饰物散布在果园中
- 轻微视差滚动效果（横向拖拽浏览果园）
- 中央下方"今日冒险"按钮始终悬浮可见

#### 答题界面

- 上方 20%：算式展示区（大字号 >= 48pt）
- 中部 60%：互动操作区（实物场景）
- 下方 20%：数字键盘或选项区（仅在需要时出现）
- 右上角：暂停按钮 + 当前进度（如 3/5）

#### 答对反馈

- 物品弹跳 + 放大缩小动画 (0.5s)
- 星星粒子爆炸效果
- 随机语音鼓励（"太棒了！" / "你真聪明！" / "继续加油！"）
- 经验值 +10 浮字

#### 答错反馈

- 温和摇头动画（不使用红色 X 或负面音效）
- 小瓢虫角色飞入，引导实物数数
- 语音："没关系，我们一起来看看吧"
- 展示辅助操作后允许重试（重试正确仍给经验值，但减半）

## 4. 技术架构

### 4.1 技术选型

| 层级 | 技术方案 | 选型理由 |
|------|---------|---------|
| 开发语言 | Swift 6 | Apple 第一方语言，性能最优 |
| UI 框架 | SwiftUI | 声明式 UI，iPad 适配便捷 |
| 游戏引擎 | SpriteKit | Apple 原生 2D 游戏框架，与 SwiftUI 无缝集成 |
| UI 动效 | Lottie (lottie-ios) | 设计师导出 After Effects 动画直接使用 |
| 音频 | AVFoundation | 背景音乐 + 即时音效，支持多轨混音 |
| 本地存储 | SwiftData | iOS 17+ 原生 ORM，替代 Core Data |
| 云同步 | CloudKit | iCloud 原生同步，家庭共享支持 |
| 图表 | Swift Charts | 家长报告数据可视化 |
| 最低版本 | iPadOS 17.0 | 覆盖 95%+ 在用 iPad |

```
NumberOrchard/

│   └── AppCoordinator.swift            # 导航协调器
```

```swift
// 核心数据模型
struct LearningProfile {
    var currentLevel: DifficultyLevel     // L1-L6
    var subDifficulty: Int                // 级别内子难度 1-5
    var consecutiveCorrect: Int
    var consecutiveWrong: Int
    var levelAccuracy: Double             // 当前级别累计正确率
    var levelQuestionCount: Int           // 当前级别累计做题量
    var averageResponseTime: TimeInterval
    var hintUsageRate: Double
}

enum DifficultyLevel: Int, Comparable {
    case seed = 1       // 5 以内加法
    case sprout = 2     // 5 以内加减法
    case smallTree = 3  // 10 以内加法
    case bigTree = 4    // 10 以内加减法
    case bloom = 5      // 20 以内加法
    case harvest = 6    // 20 以内加减法
}

// 出题策略
protocol QuestionStrategy {
    func generate(for profile: LearningProfile) -> MathQuestion
}

struct MathQuestion {
    let operand1: Int
    let operand2: Int
    let operation: Operation     // .add, .subtract
    let presentation: PresentationType  // .pickFruit, .shareFruit, .train, .balance
    let showHint: Bool
}
```

## 5. Apple Kids Category 合规

### 5.1 必须满足

| 要求 | 实施方案 |
|------|---------|
| 年龄分级 | 选择 "Ages 5 and Under" |
| 隐私保护 | 不收集任何个人信息，不集成第三方 SDK（无 Firebase Analytics 等） |
| 无第三方广告 | 永不展示广告 |
| Parental Gate | 所有付费操作/外部链接/家长中心 需通过家长门禁 |
| 无社交功能 | 无聊天、好友、排行榜等社交功能 |
| 无外部链接 | 儿童可访问区域无任何跳转外部的链接 |

### 5.2 Parental Gate 实现

```
触发场景：
    - 进入家长中心
    - 进行订阅/恢复购买操作
    - 修改任何设置

门禁方式（二选一，随机出现）：
    1. 长按圆环 3 秒 + 向右滑动
    2. 显示一道成人级数学题（如 "47 × 3 = ?"）

要求：
    - 不使用文字指引（因目标用户可能不识字）
    - 对幼儿足够困难，但对成人简单
    - 超时自动关闭（30 秒无操作）
```

### 教育理论基础

| 理论 | 应用 |
|------|------|
| CPA 教学法 (Concrete → Pictorial → Abstract) | 实物拖拽 → 图示辅助 → 纯数字算式 |
| 建构主义学习 | 孩子通过动手操作主动构建数学概念 |
| 心流理论 | 自适应难度保持在"略有挑战"区间 |
| 间隔重复 | 复习题(30%) + 新知识题(70%) 混合出题 |
| 正向强化 | 只奖励不惩罚，错误视为学习机会 |
